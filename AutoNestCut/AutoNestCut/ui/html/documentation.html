Overview

AutoNestCut is a professional SketchUp extension designed for woodworkers, furniture makers, and architects who need to optimize material usage and generate cutting diagrams for sheet goods. The extension automatically analyzes your 3D model components and creates efficient nesting layouts to minimize waste and reduce costs.

Key Features

Automatic Part Detection

Intelligently identifies sheet good components in your model based on dimensions and thickness.

Material Management

Configure stock materials with custom dimensions and pricing for accurate cost calculations.

Optimized Nesting

Advanced algorithms minimize waste by efficiently arranging parts on stock sheets.

Visual Diagrams

Generate clear cutting diagrams with part labels and dimensions for workshop use.

Detailed Reports

Comprehensive reports with material lists, costs, and efficiency metrics.

Export Options

Export to PDF, CSV, or interactive HTML formats for sharing and documentation.

How to Use AutoNestCut

Kitchen Cabinet Model Example
(Image not found: kitchen_cabinets_model_showcase.JPG)

Example: Kitchen cabinet model ready for cut list generation

  1. Prepare Your Model: Create your furniture or project in SketchUp using components for each part. Ensure sheet good parts have realistic thickness (3-50mm) and sufficient area.
  2. Select Components: Select the components or groups you want to include in your cutting list. You can select individual parts or entire assemblies.
  3. Launch Extension: Go to Extensions > AutoNestCut or click the toolbar button to open the configuration window.
  4. Configure Settings: Set your kerf width (saw blade thickness) and choose whether to allow part rotation for better nesting.
  5. Setup Materials: Add or modify stock materials with their dimensions and prices. The extension will automatically detect materials from your model.
    Stock Materials Configuration

  6. Review Parts: Check the parts preview to ensure all components are detected correctly with proper dimensions.
  7. Generate Report: Click "Generate Cut List" to process the nesting optimization and switch to the report tab.
  8. Review Results: Examine the cutting diagrams and detailed reports. Add project information like name, address, and date.

Configuration Options

General Settings

  • Kerf Width: The thickness of your saw blade cut (typically 2-4mm). This ensures proper spacing between parts.
  • Allow Rotation: Permits the algorithm to rotate parts for better nesting efficiency, unless grain direction is specified.

Stock Materials

  • Material Name: Descriptive name for the material (e.g., "Plywood_19mm", "Oak_Veneer")
  • Dimensions: Width and height of standard stock sheets in millimeters
  • Price: Cost per sheet for accurate project costing

Tips for Best Results

  • Model parts as separate components with realistic dimensions
  • Use consistent naming conventions for similar parts
  • Set appropriate grain direction attributes for wood parts
  • Verify material assignments before generating reports
  • Consider kerf width when setting up your stock materials
  • Review the parts preview to catch any detection issues
  • Performance: For large models, select only the components you need to process rather than the entire model

Troubleshooting

Parts Not Detected

  • Ensure parts are modeled as components, not loose geometry
  • Check that thickness is between 3-50mm
  • Verify minimum area requirements are met

Poor Nesting Results

  • Enable rotation if grain direction allows
  • Adjust stock sheet sizes to better match your parts
  • Consider breaking large assemblies into individual parts

Performance Issues

  • Slow Processing: The extension now includes automatic performance optimizations that reduce processing time by up to 90%
  • Memory Usage: Caching systems are automatically cleared between sessions to prevent memory buildup
  • Large Models: Select specific components rather than entire complex assemblies for faster processing
  • Extension Reload: If experiencing issues after updates, reload the extension using the Ruby Console
